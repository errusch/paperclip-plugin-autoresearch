# `awesome-paperclip` Submission Notes

This is the shortest path from the current Autoresearch work to something worth submitting to [`awesome-paperclip`](https://github.com/gsxdsm/awesome-paperclip).

## Minimum Bar

Per the current contribution guide, the project needs to be:

- relevant
- public
- maintained
- non-duplicative
- working

## Recommended Repo Name

`paperclip-plugin-autoresearch`

## Recommended One-Line Description

If submitted as a true plugin:

`Autoresearch work surface for Paperclip with bounded improvement loops, winner tracking, and local-vs-upstream runtime modes.`

If submitted earlier as a plugin candidate:

`Autoresearch control surface and runtime model for Paperclip, built as a plugin candidate ahead of the plugin runtime.`

## Suggested PR Title

`Add paperclip-plugin-autoresearch to Plugins`

## Before Submission

Make sure the public repo has:

1. a real README
2. at least one screenshot or GIF
3. a truthful install story
4. license clarity
5. current status language that does not oversell the runtime
